# AI Agent Guidelines

本ドキュメントは、マインクラフトサーバー管理WEBアプリの開発に参加するAIエージェントおよび開発者向けのガイドラインです。すべてのコードおよび設計は、以下のルールに従って作成・修正してください。

## 1. コーディング規約 (Coding Standards)

### Frontend (SvelteKit + TypeScript + TailwindCSS)
- **コンポーネントの分割**: ロジックが複雑になる場合は、適切な粒度でSvelteコンポーネントを分割すること。
- **TypeScriptの徹底**: `any` 型の使用を極力避け、適切なインターフェースや型定義を行うこと。
- **状態管理**: 状態（Stores等）はグローバルに散らかさず、必要最小限のスコープにとどめること。
- **スタイル（TailwindCSS）**: インラインスタイルや外部CSSファイルではなく、原則としてTailwindCSSのユーティリティクラスを使用すること。カスタムCSSが必要な場合も、Vaporwaveテーマに沿った一貫性を保つこと。

### Backend (Go + Echo)
- **パッケージ構成**: ハンドラー、サービス、リポジトリ層などに責務を分離し、保守性の高い構成にすること。
- **エラーハンドリング**: エラーは握りつぶさず、必ず適切にラップし、Echoのエラーミドルウェアやレスポンスに反映させること。
- **JSONのレスポンス**: フロントエンドとのやり取りはJSON形式で行い、キーはスネークケースまたはキャメルケースで統一すること（APIの設計書・規約に基づく）。
- **SQL操作**: SQLiteの操作には標準の `database/sql` もしくは適切なクエリビルダ/ORMを使用し、SQLインジェクション対策を徹底すること。

## 2. デザインルール (Vaporwave Aesthetic)

フロントエンドは「Vaporwave」のスタイルを採用します。以下の標準的なデザインルールとカラーパレットに従ってください。

### 基本カラーパレット
- **Neon Pink**: `#FF00FF` / `#FF71CE` (アクセントカラー、ボタン、エラーメッセージ等)
- **Cyan/Aqua**: `#01CDFE` / `#05FFA1` (リンク、アクティブなタブ、成功メッセージ等)
- **Purple**: `#B967FF` / `#4B0082` (背景色やヘッダー等のベースカラー)
- **Dark Blue/Black**: `#000022` / `#1A1A2E` (メインの背景色やダークモード要素)

### UI/UXのガイドライン
- **フォント**: レトロなピクセルフォントや、太字のサンセリフ体（可能であればグリッチエフェクトやドロップシャドウを適用）。
- **要素の装飾**: ネオンサインのような発光効果（`box-shadow`, `text-shadow`）を積極的に使用する。
- **レイアウト**: 1980年代〜90年代のコンピューターUIやVHS風のエフェクトを取り入れた、レトロフューチャーな構成にする。
- TailwindCSSでは、`tailwind.config.js`にこれらのカスタムカラーやフォントを登録して使用すること。

## 3. テスト駆動開発 (TDD - Test-Driven Development)

本プロジェクトではTDDを推奨します。新機能の実装やバグ修正の際は、以下のプロセスに従ってください。

1. **テストを先に書く (Red)**
   - 失敗するテストケースを作成する。
   - バックエンド: Goの標準 `testing` パッケージを使用。
   - フロントエンド: Svelte Testing LibraryやVitest（または同等のツール）を使用。
2. **実装する (Green)**
   - テストを通すための最小限のコードを実装する。
3. **リファクタリングする (Refactor)**
   - テストが通る状態を維持したまま、コードを整理・最適化する。

**特記事項**: マインクラフトサーバーのプロセス操作（起動、停止等）はモック化し、テスト環境で実際にサーバープロセスが予期せず起動しないように注意すること。
